Overview

Why I built it

Shopping around for the best price usually means opening a dozen tabs. Deal Hunter was my attempt at solving that with a single search box: type a product, and the app queries multiple sources, normalizes the results, and ranks them so the cheapest genuine listing surfaces first. I built it as a Flask backend with a lightweight, responsive front end, focused on making the core search-to-result flow feel instant.

Features

What it does

🔍

Smart product search

A single search bar handles fuzzy product matching so results stay relevant even with loose queries.

⚡

Ranked price results

Listings are normalized and sorted so the best genuine price is always the first thing you see.

📱

Responsive interface

Built mobile-first so comparing prices works just as well on a phone as it does on a desktop.

🧱

Scalable backend design

The Flask backend is structured so new retailers or data sources can be added without reworking the core logic.

How it works

Search to result

User submits a search

The query is captured from the front end and passed to the Flask backend for processing.

Backend gathers listings

The server collects matching product listings and normalizes titles, prices, and availability into a common format.

Results are ranked

Listings are scored on price and reliability signals so the strongest deal is ranked first.

Results render instantly

The front end displays ranked cards in a responsive grid, built to stay readable on any screen size.

Overview

Why I built it

This project was built to get hands-on with the full lifecycle of a data-driven application: designing a relational schema, writing clean CRUD logic in Python, and connecting it to a MySQL database. It manages student records end to end — adding new students, searching and viewing records, updating details, and removing entries — while keeping the data layer and logic cleanly separated.

Features

What it does

➕

Create records

Add new student entries with validated fields before they're written to the database.

📋

Read & search

Browse and search the full student list, with results filtered in real time.

✏️

Update details

Edit existing student records in place without disrupting related data.

🗑️

Delete records

Remove records safely, with confirmation to prevent accidental data loss.

How it works

Request to record

Action triggered
